2018_5_11Git

今天学习了git文件版本控制

了解一下日后工作git流程：

首先获取服务器项目 git clone 《url》

打开获取的文件 cd 《dir》

创建分支 git branch 《分支name》

进入到分支 git checkout 《分支name》

将所有文件加入 git管理缓存 git add .

将缓存内容加入新版本并且注释 git commit -m ‘对文件做了什么？’

然后上传服务器 git push origin 《分支name》

切换回主文件 git checkout master

如果确定你上传的文件没有什么bug你就可以合并到主文件里 git merge 《分支name》

上传到主文件 git push origin master

下面是一些基本命令

git status 查看状态

git reset –hard 版本号 使用哪个版本

版本号可以在 git log 日志里看 git reflog 反向查日志

git checkout – 清除缓存内容
